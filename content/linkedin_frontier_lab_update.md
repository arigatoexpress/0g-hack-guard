0guard update: the project now has a composed threat case file and a frontier experiment lab.

The core thesis is still simple: AI agents should not get to the wallet first.
0guard checks intent, calldata, policy, domain/reputation context, and exploit intelligence before any signer is asked to act.

New proof surfaces:

- `/api/threat-case-file` turns one risky agent request into a readable dossier across policy verdicts, hack signatures, reputation, wallet-alert quality gates, provenance, and 0G-ready receipts.
- `/api/experiments/frontier` ranks the next integrations: 0G Storage readback, GoPlus/Chainabuse/Forta reputation, Tenderly or BlockSec simulation, Telegram/TON wallet context, 0G Compute shadow scoring, and Mira claim verification.
- `/api/experiments/run` previews each experiment without live uploads, inference, signing, sends, social posts, bridges, swaps, payments, or raw payload resale.

That boundary matters. The product is not custody, a bridge, or a transaction bot. It is the checkpoint before those systems.
